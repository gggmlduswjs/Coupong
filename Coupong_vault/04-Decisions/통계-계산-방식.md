## 배경


        계정별 통계를 계산할 때 두 가지 방식 고려:
        1. 매번 DB에서 실시간 계산
        2. 일일 배치로 미리 계산하여 캐싱
        

## 결정 사항


        **선택:** 실시간 계산

        **이유:**
        - 현재 데이터 양이 적음 (월 60건)
        - 실시간 정확도가 중요
        - 캐싱 구조 복잡도 증가
        - 나중에 필요하면 최적화 가능
        

## 고려한 대안

1. 일일 배치 + Redis 캐싱
2. Materialized View 사용
